title: Configuration
description: Basic setup and built-in context helpers
---

Context helpers can be configured when initializing the TamboProvider.

## Basic Configuration

Configure built-in context helpers:

```tsx
import { TamboProvider } from "@tambo-ai/react";

function App() {
  return (
    <TamboProvider
      apiKey={process.env.NEXT_PUBLIC_TAMBO_API_KEY!}
      contextHelpers={{
        userTime: true, // Enable time context
        userPage: true, // Enable page context
      }}
    >
      {/* Your app components */}
    </TamboProvider>
  );
}
```

By default:

- `userTime` is **disabled**
- `userPage` is **disabled**

## Adding Custom Helpers

You can add custom context helpers alongside built-in ones:

```tsx
<TamboProvider
  contextHelpers={{
    // Built-in helpers
    userTime: true,
    userPage: false,

    // Custom helper
    userLocation: {
      enabled: true,
      run: async () => ({
        city: "San Francisco",
        country: "USA"
      })
    }
  }}
>
```

Custom helpers default to `enabled: true` if not specified.

## Message Context Structure

When helpers are enabled, they add data to messages using their key name:

```tsx
<TamboProvider
  contextHelpers={{
    userTime: true,
    session: {
      run: async () => ({
        sessionId: "abc123",
        userId: "user456"
      })
    }
  }}
>
```

Results in context:

```json
{
  "userTime": {
    "localTime": "1/15/2025, 3:30:00 PM",
    "timezone": "America/New_York"
  },
  "session": {
    "sessionId": "abc123",
    "userId": "user456"
  }
}
```
