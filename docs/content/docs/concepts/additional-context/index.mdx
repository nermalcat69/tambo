Additional Context helps you add additional information that will be sent along with the user's message. You can enable default context helpers, create custom helpers, or send metadata to provide more relevant AI responses.

## Overview

When enabled, context helpers automatically add metadata to messages without requiring any additional code in your components. This contextual information helps the AI:

- Provide time-aware responses (e.g., "Good morning" at the right time)
- Understand what page or section the user is currently viewing
- Give location-specific recommendations
- Provide more relevant help based on the user's current context
- Access custom business logic and application state

## How to Add Context

There are three ways to add additional context to messages:

### 1. Built-in Context Helpers

Tambo provides pre-built context helpers that can be enabled through configuration:

```tsx
<TamboProvider
  contextHelpers={{
    userTime: true,  // Enable time context
    userPage: true,  // Enable page context
  }}
>
```

### 2. Custom Context Helpers

Create your own context helpers to automatically include application-specific data:

```tsx
import { createContextHelper } from "@tambo-ai/react";

// Define helpers outside component to avoid recreating on each render
const profileHelper = createContextHelper(async () => ({
  userId: getCurrentUserId(),
  role: getUserRole(),
  preferences: await getUserPreferences()
}));

<TamboProvider
  contextHelpers={{
    // The key becomes the context name
    userProfile: profileHelper,

    // Or define inline
    appState: {
      run: async () => ({
        theme: getTheme(),
        version: getAppVersion()
      })
    }
  }}
>
```

### 3. Manual Context via Message Functions

Add custom context on a per-message basis using the `additionalContext` parameter:

```tsx
// Using useTamboThreadInput hook
const { submit } = useTamboThreadInput();

await submit({
  streamResponse: true,
  additionalContext: {
    customData: "specific to this message",
    userPreferences: { theme: "dark" },
  },
});
```

All three methods can be used together - context from helpers and manual context are merged when sending messages. See [Sending Messages](message-threads/sending-messages) for more details on manual context.

## Available Built-in Context Helpers

Tambo comes with several pre-built context helpers:

### userTime

Provides the user's current time and timezone information.

**Data provided:**

```json
{
  "timestamp": "2025-01-15T20:30:00.000Z"
}
```

### userPage

Provides information about the current page or route.

**Data provided:**

```json
{
  "title": "Analytics Dashboard",
  "url": "/dashboard/analytics"
}
```

## Creating Custom Context Helpers

Custom context helpers allow you to automatically include application-specific data with every message. The key you use in the `contextHelpers` object becomes the name of the context in messages.

### Example Usage:

```tsx
const userHelper = createContextHelper(async () => ({
  role: getUserRole(),
  permissions: getUserPermissions(),
  organization: getOrganization(),
  preferences: await getUserPreferences()
}));

// Usage
<TamboProvider
  contextHelpers={{
    user: userHelper,  // Context will be under "user" key
  }}
>
```

## When to Use Each Method

| Method               | Use When                                                 | Example                                    |
| -------------------- | -------------------------------------------------------- | ------------------------------------------ |
| **Built-in Helpers** | You need standard metadata like time or page info        | Enable `userTime` for scheduling features  |
| **Custom Helpers**   | You have app-wide context that should always be included | User profile, app settings, feature flags  |
| **Manual Context**   | Context is specific to a particular interaction          | Search filters, form data, temporary state |

## Best Practices

1. **Keep context lightweight** - Only include necessary information
2. **Handle errors gracefully** - Context helper errors won't block message sending
3. **Consider privacy** - Only share context users would expect
4. **Use meaningful keys** - The key name becomes the context name in messages
5. **Avoid duplication** - Don't send the same data through multiple methods
6. **Define helpers outside components** - Avoid recreating helpers on each render
